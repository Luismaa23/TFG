                          2. DESCRIPCION DEL PROYECTO
================================================================================

MenuMatch es una aplicacion web de recomendacion de menus del dia desarrollada
con Python y Streamlit. Su objetivo es ayudar a los usuarios a elegir el menu
mas adecuado segun sus preferencias dieteticas, restricciones alimentarias y
presupuesto disponible.

El sistema conecta dos tipos de actores principales:
  - Restaurantes: suben sus platos diarios a la plataforma.
  - Usuarios:     reciben recomendaciones personalizadas de menu.

La aplicacion implementa un motor de recomendacion dual:
  - Motor heuristico basado en reglas (baseline): filtrado estricto por
    restricciones dieteticas y presupuesto, con puntuacion ponderada que
    optimiza el equilibrio calorico.
  - Modelo de Machine Learning (Regresion Logistica): reordena los candidatos
    heuristicos segun la probabilidad de satisfaccion aprendida de valoraciones
    reales de usuarios.

Ambos motores operan simultameamente mediante un esquema de Test A/B (50/50)
que permite comparar su rendimiento en produccion y recopilar datos etiquetados.

Objetivo academico:
  Demostrar la viabilidad de un sistema de recomendacion de bajo coste
  computacional (heuristico) que sirva de baseline y, a traves del mecanismo
  de feedback del usuario, genere datos etiquetados para entrenar y validar
  un clasificador de Machine Learning en produccion.

================================================================================
                          6. MOTOR DE RECOMENDACION HEURISTICO
================================================================================

Modulo: utils/recomendador_heuristico.py

El motor opera en dos fases:

  FASE 1 - Filtrado estricto (Hard Rules):
    Elimina platos/combos que:
      - Superan el presupuesto maximo del usuario
      - Incumplen restricciones positivas (vegetariano, sin_gluten, bajo_sal):
        el COMBO debe cumplirlas en TODOS sus platos.
      - Contienen alergenos no deseados (sin_lactosa -> con_lactosa=False, etc.):
        el COMBO falla si CUALQUIER plato lo contiene.

  FASE 2 - Puntuacion (Soft Scoring):
    Formula simplificada (Rev. 4, por indicacion del tutor):

      score = 1.0 - desviacion_calorica - (precio_menu x 0.001)

      Donde: desviacion_calorica = |cal_menu - cal_objetivo| / cal_objetivo

      Si cal_menu < 0.60 x cal_objetivo:
        score = score x 0.1   (penalizacion drastica por deficit severo)

    Motivacion: El presupuesto actua solo como filtro absoluto. Las calorias
    dominan el 95% del score. El precio actua como desempate minimo.

  Generacion de combinaciones:
    - Normalizacion de texto con .lower().strip() y coincidencia por subcadena
      (ej: "postre" in categoria) para tolerar plurales y errores tipograficos.
    - Plantillas de menu:
        - Plato Unico: un solo plato (primero, segundo o plato principal).
        - Clasico:     1 Primero + 1 Segundo.
        - Ligero:      1 (Primero o Segundo) + 1 Postre.
        - Completo:    1 Primero + 1 Segundo + 1 (Postre o Acompanamiento).
        - Tradicional: 1 Primero + 1 Segundo + 1 Postre + 1 Acompanamiento.
    - Fallback: si no hay categorias suficientes, cada plato individual es un
      menu de 1 plato.

  Funciones clave del modulo:
    _cumple_restricciones(combo, restricciones) -> bool
    _calcular_score(combo, calorias_objetivo) -> float
    recomendar_menu_heuristico(platos, restricciones) -> dict | None
    recomendar_top_n(platos, restricciones, n=5) -> list[dict]
    generar_combinaciones_menu(platos) -> list[dict]

================================================================================
                          7. TEST A/B Y MODELO DE MACHINE LEARNING
================================================================================

Modulo ML: utils/ml_model.py
Pipeline: utils/ml_pipeline.py

El sistema implementa un esquema de Test A/B (50/50) activo en produccion:
  - Rama "heuristic": el motor clasico de reglas devuelve el mejor candidato.
  - Rama "ml": el modelo de Regresion Logistica reordena hasta 50 candidatos
    heuristicos por probabilidad de satisfaccion y selecciona el primero.

Cada recomendacion registra el campo recommendation_type ('heuristic' | 'ml')
en la tabla evaluaciones_menus, lo que permite comparar el rendimiento de ambas
ramas con los datos reales de los usuarios.

Mecanismo de recogida de datos (feedback loop):
  1. El usuario recibe una recomendacion del motor dual.
  2. Se le presenta un formulario de evaluacion (st.form) con 3 preguntas:
       a. Satisfaccion general (escala 1-5)
       b. Relacion calidad-precio (escala 1-5)
       c. Elegiriia realmente este menu? (Si/No)
  3. Al enviar, se guarda un registro en la tabla SQLite evaluaciones_menus
     con el contexto completo: platos, precio, calorias, score heuristico,
     restricciones activas, tipo de recomendacion y las 3 respuestas.
  4. Simultaneamente, se hace backup en Google Sheets (fire-and-forget, con
     try-except para no bloquear la UI si falla la API).

Feature engineering implementado en ml_pipeline.py:
  - price_ratio: precio / presupuesto_max (ratio de consumo del presupuesto)
  - satisfied:   variable objetivo binaria (1 si satisfaccion >= 4, 0 si <= 3)

Variables predictivas del modelo: calorias, price_ratio, score
Algoritmo: LogisticRegression(max_iter=1000) con StandardScaler

================================================================================
                         10. FLUJO COMPLETO DEL USUARIO (HAPPY PATH)
================================================================================

  Rol USUARIO:
    1. Accede a la app -> pantalla de Login (0_Login.py)
    2. Se registra o inicia sesion (rol: usuario)
    3. Ve el dashboard de Inicio con estadisticas del sistema
    4. Va a Restricciones -> configura presupuesto, calorias objetivo
       y restricciones dieteticas (vegetariano, sin_gluten, etc.)
    5. Va a Recomendaciones -> pulsa "Buscar Menu Optimo"
       -> el motor dual (heuristico o ML segun el Test A/B) devuelve
       el mejor combo y alternativas
    6. Evalua el menu recomendado con el formulario (satisfaccion, calidad-precio,
       si lo elegiria realmente)
       -> datos guardados en SQLite + backup en Google Sheets
    7. Opcionalmente: abre Carta Completa y hace un pedido libre
    8. Consulta su Historial de pedidos y puede valorarlos con 1-5 estrellas

  Rol RESTAURANTE:
    1. Inicia sesion (rol: restaurante)
    2. Va a Cargar Menu -> sube platos via formulario manual o CSV
       (nombre, precio, calorias, categoria, 6 restricciones booleanas)
    3. Los platos quedan disponibles para todos los usuarios del sistema

  Rol ADMIN:
    1. Tiene acceso a todas las paginas anteriores
    2. Panel Admin: dividido en 3 pestanas:
       - Gestion de Datos (exportacion a CSV desde Cloud/Local)
       - Inteligencia Artificial (visualizacion del dataset y reentrenamiento ML)
       - Configuracion y Logs (CRUD completo de usuarios)
